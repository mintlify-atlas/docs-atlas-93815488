---
title: "Adding a USB Device Driver"
description: "How to write a vendor-specific USB HID driver that normalizes controller reports to input_event_t and submits them to the router."
---

USB controllers communicate through a `DeviceInterface` registered in `hid_registry.c`. The registry selects the right driver for each connected device by calling each driver's `is_device()` function with the USB VID and PID. When a match is found, the driver's `init()`, `process()`, and `unmount()` functions handle the device lifecycle.

## When to add a vendor driver

The generic HID driver (`src/usb/usbh/hid/devices/generic/hid_gamepad.h`) handles most standard HID gamepads. Add a vendor-specific driver when:

- The controller uses a non-standard report format that the generic parser cannot handle (e.g., Sony DualShock 3 uses Sixaxis reports).
- The controller requires an initialization sequence before it starts sending data.
- The controller sends sensor data, touchpad data, or pressure-sensitive buttons that you want to expose.
- The controller needs custom rumble or LED output.

## File location

```
src/usb/usbh/hid/devices/vendors/<vendor>/<device>.c
src/usb/usbh/hid/devices/vendors/<vendor>/<device>.h
```

Existing vendor directories: `sony/`, `8bitdo/`, `nintendo/`, `hori/`, `logitech/`, `sega/`, `google/`, `raphnet/`, `microsoft/`, `misc/`.

## The DeviceInterface struct

Your driver exposes a `DeviceInterface` (defined in `src/usb/usbh/hid/hid_device.h`):

```c
typedef struct {
    const char* name;

    // Device identification
    bool (*is_device)(uint16_t vid, uint16_t pid);
    bool (*check_descriptor)(uint8_t dev_addr, uint8_t instance,
                             uint8_t const* desc_report, uint16_t desc_len);

    // Input processing
    void (*process)(uint8_t dev_addr, uint8_t instance,
                    const uint8_t* report, uint16_t len);

    // Output/feedback task (called periodically)
    void (*task)(uint8_t dev_addr, uint8_t instance, device_output_config_t* config);

    // Lifecycle
    bool (*init)(uint8_t dev_addr, uint8_t instance);
    void (*unmount)(uint8_t dev_addr, uint8_t instance);

    // Capabilities (optional, NULL = unknown)
    uint16_t (*get_capabilities)(void);
} DeviceInterface;
```

Most drivers only implement `is_device`, `init`, `process`, and `unmount`. The `check_descriptor` and `get_capabilities` fields are optional.

## Implementing the four required functions

### is_device

Returns `true` when the VID/PID matches a device this driver handles. List all VID/PID pairs you want to support. The registry calls this for every connected USB device.

```c
bool is_acme_pad(uint16_t vid, uint16_t pid)
{
    return (vid == 0x1234 && pid == 0x5678)  // ACME Gamepad v1
        || (vid == 0x1234 && pid == 0x5679); // ACME Gamepad v2
}
```

### init

Called once when the device first connects. Use this to send any required initialization packets, allocate per-device state, and register the device for auth passthrough if needed.

```c
bool acme_pad_init(uint8_t dev_addr, uint8_t instance)
{
    // Initialize per-device state
    acme_devices[dev_addr].rumble = 0;
    acme_devices[dev_addr].player = 0;
    printf("[acme] Connected: dev_addr=%d instance=%d\n", dev_addr, instance);
    return true;
}
```

### process

Called every time TinyUSB delivers a new HID report from the device. This is the core of the driver: parse the raw report bytes, map the vendor buttons to `JP_BUTTON_*` constants, normalize analog axes, and call `router_submit_input()`.

```c
void acme_pad_process(uint8_t dev_addr, uint8_t instance,
                      const uint8_t* report, uint16_t len)
{
    if (len < sizeof(acme_report_t)) return;

    const acme_report_t* rpt = (const acme_report_t*)report;

    // Map buttons to JP_BUTTON_* (W3C Gamepad API order)
    uint32_t buttons = 0;
    if (rpt->cross)    buttons |= JP_BUTTON_B1;
    if (rpt->circle)   buttons |= JP_BUTTON_B2;
    if (rpt->square)   buttons |= JP_BUTTON_B3;
    if (rpt->triangle) buttons |= JP_BUTTON_B4;
    if (rpt->l1)       buttons |= JP_BUTTON_L1;
    if (rpt->r1)       buttons |= JP_BUTTON_R1;
    if (rpt->l2_btn)   buttons |= JP_BUTTON_L2;
    if (rpt->r2_btn)   buttons |= JP_BUTTON_R2;
    if (rpt->select)   buttons |= JP_BUTTON_S1;
    if (rpt->start)    buttons |= JP_BUTTON_S2;
    if (rpt->l3)       buttons |= JP_BUTTON_L3;
    if (rpt->r3)       buttons |= JP_BUTTON_R3;
    if (rpt->home)     buttons |= JP_BUTTON_A1;

    // D-pad from hat switch (0=N, 1=NE, 2=E, 3=SE, 4=S, 5=SW, 6=W, 7=NW, 8=released)
    if (rpt->dpad == 0 || rpt->dpad == 1 || rpt->dpad == 7) buttons |= JP_BUTTON_DU;
    if (rpt->dpad >= 1 && rpt->dpad <= 3)                   buttons |= JP_BUTTON_DR;
    if (rpt->dpad >= 3 && rpt->dpad <= 5)                   buttons |= JP_BUTTON_DD;
    if (rpt->dpad >= 5 && rpt->dpad <= 7)                   buttons |= JP_BUTTON_DL;

    // Build and submit input event
    input_event_t event;
    init_input_event(&event);

    event.dev_addr = dev_addr;
    event.instance = instance;
    event.type = INPUT_TYPE_GAMEPAD;
    event.transport = INPUT_TRANSPORT_USB;
    event.buttons = buttons;

    // Analog axes: 0-255, sticks centered at 128, triggers at 0
    // HID convention: Y-axis 0=up, 255=down — no inversion needed for HID devices
    event.analog[ANALOG_LX] = rpt->lx;
    event.analog[ANALOG_LY] = rpt->ly;
    event.analog[ANALOG_RX] = rpt->rx;
    event.analog[ANALOG_RY] = rpt->ry;
    event.analog[ANALOG_L2] = rpt->l2_trigger;
    event.analog[ANALOG_R2] = rpt->r2_trigger;

    router_submit_input(&event);
}
```

<Warning>
  All analog axes must use HID convention: **0 = up/left, 128 = center, 255 = down/right**. Sony, Xbox, and most HID devices already use this. If a controller uses a non-HID Y-axis (like Nintendo: 0 = down, 255 = up), invert it: `event.analog[ANALOG_LY] = 255 - rpt->ly;`.
</Warning>

### unmount

Called when the device disconnects. Clean up per-device state.

```c
void acme_pad_unmount(uint8_t dev_addr, uint8_t instance)
{
    acme_devices[dev_addr].rumble = 0;
    acme_devices[dev_addr].player = 0;
    printf("[acme] Disconnected: dev_addr=%d\n", dev_addr);
}
```

## Button mapping convention

All buttons must map to `JP_BUTTON_*` constants from `src/core/buttons.h`. These follow W3C Gamepad API order:

| Constant | W3C index | Xbox | PlayStation | Switch |
|---|---|---|---|---|
| `JP_BUTTON_B1` | 0 | A | Cross | B |
| `JP_BUTTON_B2` | 1 | B | Circle | A |
| `JP_BUTTON_B3` | 2 | X | Square | Y |
| `JP_BUTTON_B4` | 3 | Y | Triangle | X |
| `JP_BUTTON_L1` | 4 | LB | L1 | L |
| `JP_BUTTON_R1` | 5 | RB | R1 | R |
| `JP_BUTTON_L2` | 6 | LT | L2 | ZL |
| `JP_BUTTON_R2` | 7 | RT | R2 | ZR |
| `JP_BUTTON_S1` | 8 | Back | Select | − |
| `JP_BUTTON_S2` | 9 | Start | Start | + |
| `JP_BUTTON_L3` | 10 | LS | L3 | LS |
| `JP_BUTTON_R3` | 11 | RS | R3 | RS |
| `JP_BUTTON_DU` | 12 | D-Up | D-Up | D-Up |
| `JP_BUTTON_DD` | 13 | D-Down | D-Down | D-Down |
| `JP_BUTTON_DL` | 14 | D-Left | D-Left | D-Left |
| `JP_BUTTON_DR` | 15 | D-Right | D-Right | D-Right |
| `JP_BUTTON_A1` | 16 | Guide | PS | Home |

Do not invent new constants. Old code may reference `USBR_BUTTON_*` — these are legacy names and must not be used in new drivers.

## Registering the driver

Add your driver to `src/usb/usbh/hid/hid_registry.c`:

<Steps>
  <Step title="Add the include">
    ```c
    #include "devices/vendors/acme/acme_pad.h"
    ```
  </Step>

  <Step title="Add a controller type constant">
    Add `CONTROLLER_ACME_PAD` to the `controller_type_t` enum in `hid_registry.h`, before `CONTROLLER_TYPE_COUNT`.
  </Step>

  <Step title="Register in register_devices()">
    ```c
    void register_devices() {
        // … existing registrations …
        device_interfaces[CONTROLLER_ACME_PAD] = &acme_pad_interface;
    }
    ```

    Vendor-specific drivers are checked before the generic fallback. The first `is_device()` that returns `true` wins.
  </Step>
</Steps>

## Full driver example

Here is a minimal but complete driver modeled on the existing Sony DS4 driver in `src/usb/usbh/hid/devices/vendors/sony/sony_ds4.c`:

<CodeGroup>

```c acme_pad.c
#include "acme_pad.h"
#include "core/buttons.h"
#include "core/router/router.h"
#include "core/input_event.h"
#include <string.h>
#include <stdio.h>

typedef struct {
    uint8_t lx, ly, rx, ry;
    struct {
        uint8_t dpad   : 4;
        uint8_t square : 1;
        uint8_t cross  : 1;
        uint8_t circle : 1;
        uint8_t tri    : 1;
    };
    struct {
        uint8_t l1     : 1;
        uint8_t r1     : 1;
        uint8_t l2_btn : 1;
        uint8_t r2_btn : 1;
        uint8_t select : 1;
        uint8_t start  : 1;
        uint8_t l3     : 1;
        uint8_t r3     : 1;
    };
    uint8_t l2_trigger;
    uint8_t r2_trigger;
} __attribute__((packed)) acme_report_t;

bool is_acme_pad(uint16_t vid, uint16_t pid)
{
    return (vid == 0x1234 && pid == 0x5678);
}

bool acme_pad_init(uint8_t dev_addr, uint8_t instance)
{
    printf("[acme] init dev_addr=%d\n", dev_addr);
    return true;
}

void acme_pad_process(uint8_t dev_addr, uint8_t instance,
                      const uint8_t* report, uint16_t len)
{
    if (len < sizeof(acme_report_t)) return;
    const acme_report_t* rpt = (const acme_report_t*)report;

    uint32_t buttons = 0;
    if (rpt->cross)   buttons |= JP_BUTTON_B1;
    if (rpt->circle)  buttons |= JP_BUTTON_B2;
    if (rpt->square)  buttons |= JP_BUTTON_B3;
    if (rpt->tri)     buttons |= JP_BUTTON_B4;
    if (rpt->l1)      buttons |= JP_BUTTON_L1;
    if (rpt->r1)      buttons |= JP_BUTTON_R1;
    if (rpt->l2_btn)  buttons |= JP_BUTTON_L2;
    if (rpt->r2_btn)  buttons |= JP_BUTTON_R2;
    if (rpt->select)  buttons |= JP_BUTTON_S1;
    if (rpt->start)   buttons |= JP_BUTTON_S2;
    if (rpt->l3)      buttons |= JP_BUTTON_L3;
    if (rpt->r3)      buttons |= JP_BUTTON_R3;

    if (rpt->dpad == 0 || rpt->dpad == 1 || rpt->dpad == 7) buttons |= JP_BUTTON_DU;
    if (rpt->dpad >= 1 && rpt->dpad <= 3)                   buttons |= JP_BUTTON_DR;
    if (rpt->dpad >= 3 && rpt->dpad <= 5)                   buttons |= JP_BUTTON_DD;
    if (rpt->dpad >= 5 && rpt->dpad <= 7)                   buttons |= JP_BUTTON_DL;

    input_event_t event;
    init_input_event(&event);
    event.dev_addr = dev_addr;
    event.instance = instance;
    event.type = INPUT_TYPE_GAMEPAD;
    event.transport = INPUT_TRANSPORT_USB;
    event.buttons = buttons;
    event.analog[ANALOG_LX] = rpt->lx;
    event.analog[ANALOG_LY] = rpt->ly;
    event.analog[ANALOG_RX] = rpt->rx;
    event.analog[ANALOG_RY] = rpt->ry;
    event.analog[ANALOG_L2] = rpt->l2_trigger;
    event.analog[ANALOG_R2] = rpt->r2_trigger;

    router_submit_input(&event);
}

void acme_pad_unmount(uint8_t dev_addr, uint8_t instance)
{
    printf("[acme] disconnected dev_addr=%d\n", dev_addr);
}

DeviceInterface acme_pad_interface = {
    .name       = "ACME Gamepad",
    .is_device  = is_acme_pad,
    .init       = acme_pad_init,
    .process    = acme_pad_process,
    .unmount    = acme_pad_unmount,
};
```

```c acme_pad.h
#ifndef ACME_PAD_H
#define ACME_PAD_H

#include "usb/usbh/hid/hid_device.h"

extern DeviceInterface acme_pad_interface;

#endif // ACME_PAD_H
```
