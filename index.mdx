Joypad OS is firmware that runs on small microcontroller boards. Flash it onto a chip and it handles everything: reading controllers, translating protocols, routing inputs to outputs, and managing button remapping. Use any modern USB or Bluetooth controller on retro consoles, or build your own custom controller from scratch.

## How it works

Joypad OS normalizes every input source into a common format, routes it through configurable player slots, and translates it into the output protocol the target needs — from retro console protocols via PIO to modern USB HID and XInput.

```
Any Input                Router              Any Output
─────────                ──────              ──────────
USB controllers ──┐                          ┌──→ Retro consoles
Bluetooth ────────┤  normalize → route   ───┼──→ USB gamepad
WiFi ─────────────┼──→  → translate ────────┼──→ XInput / PS4 / Switch
SNES / N64 / GC ──┤         ↓               ├──→ Keyboard + mouse
GPIO buttons ─────┘    profile_apply()       └──→ Custom outputs
```

## Supported consoles and controllers

**Output consoles**: GameCube/Wii, PCEngine/TurboGrafx-16, Dreamcast, Nuon, 3DO, N64, Casio Loopy, Neo Geo, NES

**Input controllers**: Xbox (OG, 360, One, Series X|S), PlayStation (DS3, DS4, DualSense), Nintendo Switch Pro, Joy-Con, 8BitDo, Logitech, Hori, Google Stadia, and generic USB HID gamepads

**Platforms**: RP2040 (Adafruit KB2040, Raspberry Pi Pico/W, Waveshare RP2040-Zero), ESP32-S3 (XIAO ESP32-S3), nRF52840 (Seeed XIAO, Adafruit Feather)
