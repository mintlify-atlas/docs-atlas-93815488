All input drivers normalize controller buttons to the `JP_BUTTON_*` constants defined in `src/core/buttons.h`. These follow the W3C Gamepad API standard ordering: bit position equals the W3C button index, so `1 << index` gives the bitmask directly.

## Controller layout

```
           ____________________________              __
          / [__L2__]          [__R2__] \               |
         / [__ L1 __]        [__ R1 __] \              | Triggers
      __/________________________________\__         __|
     /                                  _   \          |
    /      /\           __             (B4)  \         |
   /       ||      __  |A1|  __     _       _ \        | Main Pad
  |    <===DP===> |S1|      |S2|  (B3) -|- (B2)|       |
   \       ||      --        --       _       /        |
   /\      \/   /   \        /   \   (B1)   /\       __|
  /  \________ | LS  | ____ | RS  | _______/  \        |
 |         /  \ \___/ /    \ \___/ /  \         |      | Sticks
 |        /    \_____/      \_____/    \        |    __|
 |       /       L3            R3       \       |
  \_____/                                \_____/
```

## Button definitions

Defined in `src/core/buttons.h`:

```c
// Face buttons (right cluster)
#define JP_BUTTON_B1 (1 << 0)   // A         B         Cross      2
#define JP_BUTTON_B2 (1 << 1)   // B         A         Circle     3
#define JP_BUTTON_B3 (1 << 2)   // X         Y         Square     1
#define JP_BUTTON_B4 (1 << 3)   // Y         X         Triangle   4

// Shoulder buttons
#define JP_BUTTON_L1 (1 << 4)   // LB        L         L1         5
#define JP_BUTTON_R1 (1 << 5)   // RB        R         R1         6
#define JP_BUTTON_L2 (1 << 6)   // LT        ZL        L2         7
#define JP_BUTTON_R2 (1 << 7)   // RT        ZR        R2         8

// Center cluster
#define JP_BUTTON_S1 (1 << 8)   // Back      -         Select     9
#define JP_BUTTON_S2 (1 << 9)   // Start     +         Start      10

// Stick clicks
#define JP_BUTTON_L3 (1 << 10)  // LS        LS        L3         11
#define JP_BUTTON_R3 (1 << 11)  // RS        RS        R3         12

// D-pad
#define JP_BUTTON_DU (1 << 12)  // D-Up      D-Up      D-Up       Hat
#define JP_BUTTON_DD (1 << 13)  // D-Down    D-Down    D-Down     Hat
#define JP_BUTTON_DL (1 << 14)  // D-Left    D-Left    D-Left     Hat
#define JP_BUTTON_DR (1 << 15)  // D-Right   D-Right   D-Right    Hat

// Auxiliary
#define JP_BUTTON_A1 (1 << 16)  // Guide     Home      PS         13
#define JP_BUTTON_A2 (1 << 17)  // -         Capture   Touchpad   14
#define JP_BUTTON_A3 (1 << 18)  // -         -         Mute       -
#define JP_BUTTON_A4 (1 << 19)  // -         -         -          -

// Paddles (extended)
#define JP_BUTTON_L4 (1 << 20)  // P1        -         -          -
#define JP_BUTTON_R4 (1 << 21)  // P2        -         -          -
```

Buttons are active-high: bit = 1 means pressed, 0 means released.

## Analog axes

Defined in `src/core/input_event.h` as `analog_axis_index_t`. All values are 0–255. Sticks are centered at 128; triggers and spinner start at 0.

```c
typedef enum {
    ANALOG_LX = 0,   // Left stick X  (0=left,    128=center, 255=right)
    ANALOG_LY = 1,   // Left stick Y  (0=up,       128=center, 255=down)
    ANALOG_RX = 2,   // Right stick X (0=left,    128=center, 255=right)
    ANALOG_RY = 3,   // Right stick Y (0=up,       128=center, 255=down)
    ANALOG_L2 = 4,   // Left trigger  (0=released, 255=fully pressed)
    ANALOG_R2 = 5,   // Right trigger (0=released, 255=fully pressed)
    ANALOG_RZ = 6,   // RZ axis / twist / spinner
    ANALOG_COUNT = 7,
} analog_axis_index_t;
```

<Warning>
**Y-axis convention (HID standard):** 0 = stick pushed UP, 128 = centered, 255 = stick pushed DOWN. This matches USB HID and DirectInput.

Nintendo controllers use the opposite convention (0 = down, 255 = up). Input drivers for Nintendo hardware **must** invert the Y axis before submitting events to the router. Native GC/N64 host drivers also invert Y when reading.
</Warning>

| Index | Constant | Range | Default | Description |
|-------|----------|-------|---------|-------------|
| 0 | `ANALOG_LX` | 0–255 | 128 | Left stick X (0=left, 255=right) |
| 1 | `ANALOG_LY` | 0–255 | 128 | Left stick Y (0=up, 255=down) |
| 2 | `ANALOG_RX` | 0–255 | 128 | Right stick X (0=left, 255=right) |
| 3 | `ANALOG_RY` | 0–255 | 128 | Right stick Y (0=up, 255=down) |
| 4 | `ANALOG_L2` | 0–255 | 0 | Left trigger (0=released, 255=fully pressed) |
| 5 | `ANALOG_R2` | 0–255 | 0 | Right trigger (0=released, 255=fully pressed) |
| 6 | `ANALOG_RZ` | 0–255 | 0 | RZ axis / twist / spinner |

## Controller layout classification

The `controller_layout_t` enum describes the physical button arrangement. Relevant primarily for 6-button controllers where row position determines the output mapping.

```c
typedef enum {
    LAYOUT_UNKNOWN = 0,      // Unknown or default (4-face button modern gamepad)
    LAYOUT_MODERN_4FACE,     // SNES/PlayStation style
    LAYOUT_NINTENDO_4FACE,   // Nintendo BAYX face style
    LAYOUT_NINTENDO_N64,     // A/B + C-buttons + Z
    LAYOUT_GAMECUBE,         // AXBY face style
    LAYOUT_SEGA_6BUTTON,     // Genesis/Saturn: Bottom [A][B][C], Top [X][Y][Z]
    LAYOUT_PCE_6BUTTON,      // PCEngine: Bottom [III][II][I], Top [IV][V][VI]
    LAYOUT_ASTROCITY,        // Astrocity: Bottom [D][E][F], Top [A][B][C]
    LAYOUT_3DO_3BUTTON,      // 3DO: Single row [A][B][C]
} controller_layout_t;
```

### GP2040-CE canonical mapping

All 6-button layouts use the GP2040-CE positional mapping internally:

```
  Top row:    [B3][B4][R1]
  Bottom row: [B1][B2][R2]
```

Layout transform functions (e.g., `transform_to_pce_layout()`) convert between layouts when required by output drivers.

| Layout | Physical arrangement | Examples |
|--------|---------------------|----------|
| `LAYOUT_MODERN_4FACE` | Standard 4-face (SNES/PS/Xbox style) | DualSense, Xbox, Pro Controller |
| `LAYOUT_NINTENDO_4FACE` | Nintendo BAYX face style | SNES controllers |
| `LAYOUT_NINTENDO_N64` | A/B + C-buttons + Z | N64 controllers |
| `LAYOUT_GAMECUBE` | AXBY face style | GameCube controllers |
| `LAYOUT_SEGA_6BUTTON` | Bottom [A][B][C], Top [X][Y][Z] | Genesis/Saturn 6-button pads |
| `LAYOUT_PCE_6BUTTON` | Bottom [III][II][I], Top [IV][V][VI] | PCEngine Avenue Pad 6 |
| `LAYOUT_ASTROCITY` | Bottom [D][E][F], Top [A][B][C] | Sega Astro City Mini arcade |
| `LAYOUT_3DO_3BUTTON` | Single row [A][B][C] | 3DO controllers |

## Legacy names

Older code in the repository uses `USBR_BUTTON_*` names (the old `usbretro` project prefix). These are deprecated aliases for the same values. Use `JP_BUTTON_*` in all new code.
