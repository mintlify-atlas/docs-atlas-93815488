Joypad OS runs on RP2040, ESP32-S3, and nRF52840 microcontrollers. The right board depends on what you're building: console adapters, Bluetooth bridges, or compact embedded builds.

## RP2040 boards

### Adafruit KB2040

**Recommended for most builds.**

<CardGroup cols={2}>
  <Card title="Form factor" icon="microchip">
    Pro Micro compatible — 1.3" × 0.7"
  </Card>
  <Card title="Purchase" icon="cart-shopping" href="https://www.adafruit.com/product/5302">
    Adafruit — ~$10
  </Card>
</CardGroup>

**Features**: USB-C, 21 GPIO pins, boot button, WS2812 RGB LED

**Recommended apps**: USB2PCE, USB2GC, USB2Nuon, USB2DC, USB2Loopy, USB2Neo

The KB2040 is the default board for most console adapters. It uses a USB-C connector, has a built-in RGB LED for status indication, and fits the Pro Micro footprint used in existing adapter designs.

<Tip>
  The built-in RGB LED doubles as a status indicator — solid colors for connected controllers, patterns for boot and pairing states.
</Tip>

**Features**: Micro-USB (Pico) or USB-C (Pico 2 W), 26 GPIO pins, boot button

**Recommended apps**: All console adapters; `bt2usb` and `wifi2usb` on Pico W / Pico 2 W

| Variant | USB | Bluetooth | WiFi | Price |
|---------|-----|-----------|------|-------|
| Pico | Micro-USB | No | No | ~$4 |
| Pico W | Micro-USB | Classic BT + BLE | Yes | ~$6 |
| Pico 2 W | USB-C | Classic BT + BLE | Yes | ~$7 |

The **Pico W** and **Pico 2 W** have built-in Classic Bluetooth and BLE — no dongle needed for wireless controllers or WiFi apps.

### Adafruit Feather RP2040 USB Host

<CardGroup cols={2}>
  <Card title="Form factor" icon="microchip">
    Feather — 2.0" × 0.9"
  </Card>
  <Card title="Purchase" icon="cart-shopping" href="https://www.adafruit.com/product/5723">
    Adafruit — ~$12
  </Card>
</CardGroup>

**Features**: USB-C (device) + USB-A (host), 21 GPIO pins, boot button, RGB LED

**Recommended apps**: `usb2usb` (USB passthrough)

The Feather RP2040 USB Host has a built-in USB-A host port — no wiring or breakout board needed. Use it for USB passthrough applications and USB-to-USB conversion without soldering.

<Note>
  This is the only RP2040 board with a built-in USB-A port. All other RP2040 boards require wiring a USB-A connector to GPIO pins for controller input. See the [Wiring Guide](/hardware/wiring).
</Note>

**Features**: USB-C, BLE, WiFi, 11 GPIO pins, boot button, user LED (GPIO 21, active low)

<Warning>
  The XIAO ESP32-S3 supports **BLE only** — no Classic Bluetooth. Classic BT controllers such as DualShock 3, DualShock 4, DualSense, Switch Pro, and Wii U Pro Controller are not supported. Use the Pico W or a USB Bluetooth dongle for those.
</Warning>

Requires ESP-IDF to build. See the [ESP32-S3 platform docs](/platforms/esp32).

## Power requirements

RP2040 boards draw roughly 50–100 mA. Controllers vary from 50 mA (simple D-pads) to 500 mA (Xbox with rumble).

| Setup | Recommendation |
|-------|---------------|
| 1–2 controllers | Bus-powered USB hub is fine; no external power needed |
| 3+ controllers | Use a powered USB hub, especially with rumble |
| High-power devices (Xbox rumble, RGB peripherals) | Powered USB hub required |

A USB 2.0 port provides 500 mA maximum. Running multiple rumble controllers or RGB gaming peripherals on a passive hub will cause brownouts.
