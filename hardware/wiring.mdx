This guide covers how to wire USB host ports and console connectors to your board. For board-specific build walkthroughs, see the [DIY Builds](/hardware/diy) page.

## USB host port wiring

Most RP2040 boards need a USB-A connector wired to GPIO pins to accept controller input. Joypad OS uses PIO-based USB host (bit-banged via the RP2040's PIO peripheral), which allows simultaneous USB host and device — the native USB port acts as a device (for flashing and USB output modes) while PIO-USB provides the host port for controllers.

<Note>
  The **Adafruit Feather RP2040 USB Host** has a built-in USB-A port — no wiring needed. nRF52840 boards use a MAX3421E chip for USB host instead of PIO-USB.
</Note>

**PIO-USB limitations:**
- Full Speed (12 Mbps) only
- D+ and D- must be on consecutive GPIO pins (D- = D+ + 1)
- Connect your controller directly — USB hubs are not reliable on PIO-USB host due to [upstream enumeration and disconnect issues](https://github.com/sekigon-gonnoc/Pico-PIO-USB/issues/149)

### USB-A connector pinout

```
  ┌───────────────────┐
  │ 1   2   3   4     │
  └───────────────────┘
   VBUS  D-  D+  GND
   (5V)
```

| USB-A pin | Signal | Typical wire color |
|-----------|--------|--------------------|
| 1 | VBUS (+5V) | Red |
| 2 | D- | White |
| 3 | D+ | Green |
| 4 | GND | Black |

### What you need

- **USB-A female breakout board** ([example](https://www.adafruit.com/product/1833)) or a cut USB-A extension cable
- **4 jumper wires** (or 22–26 AWG wire with soldering iron)

---

### Tips

- **Double-check D+ and D-** — swapping them is the most common mistake and will silently fail
- **Keep wires short** — USB signal integrity degrades with long runs; under 15 cm is ideal
- **USB-A breakout boards** are easier than cutting cables — labeled pins reduce wiring errors
- **No hubs** — connect your controller directly to the USB-A port

---

### Troubleshooting

<AccordionGroup>
  <Accordion title="No controller detected">
    - Verify D+ and D- are not swapped
    - Check that VBUS is providing 5V to the USB-A connector
    - Try a different controller or cable
    - Confirm you flashed the correct firmware for your board
  </Accordion>
  <Accordion title="Controller disconnects or is flaky">
    - Shorten your wires
    - Check for cold solder joints
    - Some controllers draw more current than VBUS can supply — use external 5V power
  </Accordion>
  <Accordion title="Works with some controllers but not others">
    Check the [controller compatibility list](/hardware/controllers) for supported devices. Some controllers have dedicated drivers; others rely on generic HID.
  </Accordion>
</AccordionGroup>

## Console connector pinouts

### GameCube

The GameCube controller port uses a proprietary connector. Cut a GameCube extension cable and wire the console-end plug to your board.

| KB2040 pin | GC signal | Notes |
|------------|-----------|-------|
| GPIO 7 | Data | Bidirectional joybus data line |
| GPIO 6 | 3.3V sense | Connect directly to 3V3 on KB2040 (no external wire needed) |
| 3V3 | 3.3V | Powers the data line pull-up |
| GND | GND | Ground |

GPIO 6 is set HIGH to signal "play mode" (GameCube console connected). When GPIO 6 reads LOW, the adapter falls back to USB device mode.

Optional shield pins for proper shielding:

| KB2040 pin | Function |
|------------|----------|
| GPIO 4 | Shield pin L |
| GPIO 5 | Shield pin L+1 |
| GPIO 26 | Shield pin R |
| GPIO 27 | Shield pin R+1 |

**Dreamcast Maple bus connector:**

```
  Looking at controller plug end:

    1 2 3 4 5
     -------
    |       |
    +---+---+

  1 = SDCKA (Data A)
  2 = Sense (3.3V, active when console detects controller)
  3 = +5V
  4 = GND
  5 = SDCKB (Data B)
```

| KB2040 pin | DC pin | Signal | Notes |
|------------|--------|--------|-------|
| GPIO 2 | Pin 1 | Data line A | Maple bus SDCKA |
| GPIO 3 | Pin 5 | Data line B | Maple bus SDCKB (must be GPIO 2 + 1) |
| 5V | Pin 3 | +5V | Power from Dreamcast |
| GND | Pin 4 | GND | Ground |

The Dreamcast Maple bus requires two consecutive GPIO pins for the differential data lines.

## Native controller wiring

Native controllers wire directly to GPIO pins and communicate using their original console protocols.

### SNES / NES

The SNES controller port is a 7-pin connector. Cut an extension cable and wire the console-end plug to your board.

```
   ___________
  /  1 2 3 4  \
 |  5 6 7      |
  \____________/

 1 = +5V        5 = DATA (serial data from controller)
 2 = CLOCK      6 = LATCH (active-high strobe)
 3 = IOBIT      7 = GND
 4 = DATA1
```

| KB2040 GPIO | SNES pin | Signal | Direction |
|-------------|----------|--------|-----------|
| GPIO 5 | 2 | CLOCK | Output to controller |
| GPIO 6 | 6 | LATCH | Output to controller |
| GPIO 7 | 5 | DATA0 | Input from controller |
| GPIO 8 | 4 | DATA1 | Input (multitap / keyboard) |
| GPIO 9 | 3 | IOBIT | Output (mouse / keyboard) |
| 5V | 1 | +5V | Power to controller |
| GND | 7 | GND | Ground |

NES controllers are auto-detected (8 buttons + D-pad). SNES mouse requires IOBIT wiring. Xband keyboard requires DATA1 + IOBIT wiring.

---

### N64

The N64 controller uses a single-wire joybus protocol. Wire a cut N64 extension cable to your board.

| KB2040 GPIO | N64 signal | Notes |
|-------------|-----------|-------|
| GPIO 29 (A3) | Data | Bidirectional joybus data line |
| 3V3 | 3.3V | Power to controller |
| GND | GND | Ground |

GPIO 29 is used because it is the analog-capable A3 pin on the KB2040. The KB2040 runs at 130 MHz for precise joybus timing.

---

### GameCube (native input)

The GameCube native input wiring is the same connector as the console output adapter, but used in reverse — reading a GameCube controller rather than driving a console port.

| KB2040 GPIO | GC signal | Notes |
|-------------|-----------|-------|
| GPIO 29 | Data | Bidirectional joybus data line |
| 3V3 | 3.3V | Power to controller |
| GND | GND | Ground |

## General tips

**Power:**
- RP2040 boards run at 3.3V logic. Console protocols typically run at 3.3V or 5V — check the target console before wiring.
- Use VBUS (5V) to power controllers that need it; use 3V3 only for low-power devices.
- For high-current draws (rumble controllers), power the board via VSYS with an external 5V supply rather than relying on bus power from the host.

**Signal integrity:**
- Keep all signal wires under 15 cm.
- Use 22–26 AWG wire. Thinner wire increases resistance and can cause signal issues.
- Twist D+ and D- pairs together for USB runs over 5 cm.
- Check for cold solder joints if you see intermittent behavior.

**Ground:**
- Always connect grounds. A floating ground will cause erratic or no signal.
- For console connectors, connect both signal ground and shield ground where applicable.
